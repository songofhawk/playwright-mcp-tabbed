playwright-mcp-tabbed

Give every agent its own browser tab while keeping one shared login session. Built for Cursor and other MCP clients that need reliable parallel browser workflows.
Per-tool `tab_index` No shared current-tab pointer. Each tool call targets the intended tab explicitly.
Shared auth state All tabs live in the same browser context, so cookies and login state are reused.
Multi-agent friendly Ideal for batch bug fixing, parallel QA, and side-by-side verification workflows.

Typical use cases

Batch-fix ClickUp bugs, verify multiple routes after a refactor, compare two environments, or split one long browser workflow across several agents.

Why not just `browser_tabs.select`?

Because a shared active tab becomes a race condition under concurrency. Explicit `tab_index` keeps intent stable on every tool call.
